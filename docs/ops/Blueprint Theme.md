## Blueprint v0.1 — Theme / Brand / PR / Style / Tone

**Brand:** Asset Management Property (AMP)
**อ้างอิงโลโก้ที่แนบ:** โทนหลักมี **น้ำเงินเข้ม + ส้มคอรัล/แดงอมส้ม** (เอกลักษณ์ชัด, โมเดิร์น, ดู “มืออาชีพ-จริงจัง”)

---

# 1) Brand Core (แก่นแบรนด์)

## 1.1 Positioning (วางตัวในตลาด)

**“ทีมบริหารอสังหา/ตัวแทนที่ทำงานแบบมีระบบ — ข้อมูลชัด ตอบไว ปิดงานไว”**

* แตกต่างจากตลาดที่เน้นขายเก่งแต่ “ข้อมูลไม่แน่น / ตอบช้า / follow-up หลุด”

## 1.2 Brand Promise (คำมั่น)

* **ชัดเจนเรื่องข้อมูล** (ราคา/เงื่อนไข/ค่าใช้จ่าย/สถานะห้อง)
* **โปร่งใส** (อัปเดตล่าสุด, แจ้งข้อจำกัด, ไม่โฆษณาเกินจริง)
* **ตอบไว** (speed-to-lead เป็น KPI ภายใน)

## 1.3 Brand Values (ค่านิยม)

* **Evidence-first**: พูดจากข้อมูล/หลักฐาน
* **Speed with accuracy**: เร็วแต่ไม่มั่ว
* **Client-protection**: ลดความเสี่ยงโดนหลอก/ข้อมูลไม่ครบ

---

# 2) Visual Identity Blueprint (ธีม/แบรนด์)

> ตั้งใจ “คง DNA โลโก้” แล้วทำให้เว็บ/สื่อดูเป็นระบบเดียวกัน

## 2.1 Color System (จากโลโก้)

### Primary Palette

* **AMP Blue (Primary)**: `#1744BE` (น้ำเงินเด่น/ความน่าเชื่อถือ)
* **AMP Coral (Accent)**: `#F05A43` (ส้มคอรัล/พลัง/CTA)
* **White**: `#FFFFFF`
* **Ink (Text)**: `#0F172A` (อ่านง่าย)

### Neutral Palette (สำหรับ UI)

* **Gray-50** `#F8FAFC`
* **Gray-100** `#F1F5F9`
* **Gray-300** `#CBD5E1`
* **Gray-600** `#475569`

### Usage Rules (สำคัญ)

* พื้นหลังหลักใช้ **ขาว/เทาอ่อน 90%**
* น้ำเงินใช้กับ **หัวข้อ/ลิงก์/กรอบสำคัญ**
* คอรัลใช้กับ **CTA/Badge/ตัวเลขราคา/จุดเน้น** (ไม่เกิน 10–15% ของหน้า)
* หลีกเลี่ยง “พื้นหลังคอรัลทั้งบล็อกใหญ่” เพราะจะดูเร่งขาย

## 2.2 Typography (TH/EN)

### ฟอนต์แนะนำ (Google Fonts)

* **TH/EN UI + Body:** `Prompt` หรือ `Kanit` (อ่านง่าย, มืออาชีพ)
* **EN-only (ถ้าต้องการภาพลักษณ์ SaaS):** `Inter` คู่กับฟอนต์ไทย 1 ตัว

### Type Scale (เว็บอสังหา)

* H1: 28–34 (mobile 24–28)
* H2: 20–24
* Body: 16
* Caption: 12–13
* ตัวเลขราคา/ไฮไลต์: 22–28 (เน้นชัด)

## 2.3 UI Style (Look & Feel)

**สไตล์:** Modern clean + trust-first

* Card: ขอบโค้ง 12–16px, shadow เบา
* Grid: 12-column (desktop), 1-column (mobile), 2-column (tablet)
* Icon: line icon เรียบ (ไม่การ์ตูน)
* Photo: โทนธรรมชาติ, แสงจริง, ไม่ใส่ฟิลเตอร์จัด

## 2.4 Logo Usage Rules

* **ห้ามยืด/บีบ**
* เว้นระยะรอบโลโก้อย่างน้อย **1x ความสูงตัวอักษร “Asset”**
* บนพื้นหลังเข้มให้ใช้เวอร์ชันที่ยังคอนทราสต์ชัด (ถ้ามีไฟล์ inverse ให้ทำใน brand kit)
* ขนาดขั้นต่ำบนมือถือ: ความสูงโลโก้ **24px+** (กันอ่านไม่ออก)

# 3) UX Theme (การจัดวางเพื่อ Conversion)

## 3.1 Hero Message (Homepage)

**Head:** “Asset Management Property”
**Sub:** “ข้อมูลชัด ตอบไว ดูแลครบ”
**Primary CTA:** “แชท LINE ตอนนี้” (คอรัล)
**Secondary:** “ดูทรัพย์ทั้งหมด” (น้ำเงิน/outline)

## 3.2 Trust Cues (ต้องมีทุกหน้า detail)

* “อัปเดตล่าสุด: …”
* “ผู้ดูแล: … (ภาษา TH/EN)”
* “แจ้งประกาศนี้ (Report)” + anti-scam note สั้น ๆ

# 4) Tone of Voice & Writing Style (สไตล์ภาษา/โทน)

## 4.1 บุคลิกแบรนด์ (Brand Voice)

* **Professional, calm, precise**
* ไม่เล่นมุก ไม่เว่อร์ ไม่คำโฆษณาเกินจริง
* “พูดเหมือนผู้จัดการทรัพย์” ไม่ใช่ “นักขายปั่น”

## 4.2 คำที่ใช้ / ไม่ใช้

### ใช้

* “อัปเดตล่าสุด”, “เงื่อนไข”, “รวม/ไม่รวม”, “พร้อมเข้าอยู่”, “ค่าใช้จ่ายวันโอน”

### เลี่ยง

* “ถูกที่สุดในพัทยา”, “การันตีได้กำไร”, “ด่วนมาก!!!”, “หลุดจอง 100%”

## 4.3 Template ข้อความ (TH/EN)

### Listing headline (TH)

* `คอนโด 1 ห้องนอน | จอมเทียน | พร้อมอยู่ | ฿15,000/เดือน`

### Listing headline (EN)

* `1BR Condo | Jomtien | Ready to move-in | THB 15,000/mo`

### CTA microcopy

* TH: “แชทเพื่อเช็กห้องว่าง”, “ขอรูป/วิดีโอเพิ่ม”, “นัดดูห้อง”
* EN: “Check availability”, “Request more photos/video”, “Book a viewing”

# 5) PR Blueprint (ภาพลักษณ์ + ข่าว + สื่อ)

> เป้าหมาย PR ของ AMP: “ทำให้คนเชื่อว่าเป็นทีมที่มีระบบและไว้ใจได้” ไม่ใช่แค่มีทรัพย์เยอะ

## 5.1 PR Narrative (เรื่องเล่าหลัก)

* “AMP ยกระดับงานตัวแทน/บริหารอสังหาด้วยระบบข้อมูล + ความเร็วในการตอบลูกค้า”
* “ลดความเสี่ยงลูกค้าโดนหลอก/ข้อมูลไม่ครบ ด้วยการอัปเดตและตรวจสอบสถานะ”

## 5.2 Messaging Pillars (3 เสาหลัก)

1. **Accuracy:** ข้อมูลครบ-ตรง (ราคา/เงื่อนไข/ค่าใช้จ่าย)
2. **Speed:** ตอบไว (SLA ภายใน X นาที)
3. **Trust:** โปร่งใส (อัปเดตล่าสุด + ช่องทางติดต่อชัด + report)

## 5.3 PR Assets (Press Kit ที่ควรมี)

* 1. Logo pack: PNG/SVG + สีมาตรฐาน + safe area
* 2. Brand one-pager: positioning, services, coverage area
* 3. Founder/Team bio (สั้น 100–150 คำ)
* 4. Fact sheet: จำนวน listing, โซนบริการ, SLA response
* 5. Photo kit: ทีม/ออฟฟิศ/ภาพตัวอย่างทรัพย์ (เลือกคุณภาพสูง)
* 6. Boilerplate (ท้ายข่าวประชาสัมพันธ์)

## 5.4 PR Templates (พร้อมใช้)

### Announcement (สั้น)

* “AMP เปิดตัวเว็บไซต์ใหม่ amppattaya.com — ค้นหาทรัพย์ TH/EN + ติดต่อทีมได้ภายใน 1 คลิกผ่าน LINE”

### Case story (น่าเชื่อถือกว่า)

* “จากปัญหาลูกค้าหาห้องไม่เจอ/ข้อมูลไม่ครบ → AMP ทำระบบอัปเดตทรัพย์และตอบไว ช่วยลดเวลาตัดสินใจ”

# 6) Brand System for Digital (Design Tokens ใช้กับเว็บ)

> เอาไปทำเป็น CSS variables ได้ทันที (ไม่ต้องเป็นโค้ดวันนี้ แค่เป็นสัญญา)

## 6.1 Tokens (แนวทาง)

* `--color-primary: #1744BE;`
* `--color-accent: #F05A43;`
* `--radius-card: 16px;`
* `--shadow-card: subtle;`
* spacing scale: 4/8/12/16/24/32

## 6.2 Component Rules

* Primary button = **Accent (coral)** + ตัวอักษรขาว
* Secondary button = **Outline primary**
* Badge:

  * Featured = coral
  * Updated = blue
  * Available now = green (เพิ่มสีที่ 3 แบบคุมโทน: `#16A34A`)

---

# 7) “Definition of Done” สำหรับ Brand/Theme (สิ่งที่ต้องส่งมอบ)

1. Brand kit 1 หน้า (สี/ฟอนต์/โลโก้/การใช้งาน)
2. Copy kit (headline/CTA/คำต้องห้าม) TH/EN
3. UI kit ระดับ component (card, button, badge, sticky CTA)
4. Press kit (logo pack + boilerplate + one pager)
