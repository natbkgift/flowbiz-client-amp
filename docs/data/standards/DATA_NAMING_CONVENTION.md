## üìÖ Date Format Standards

### Standard Format: YYYY-MM-DD

**Always use ISO 8601 format:**

```
‚úÖ Good:  2026-01-15
‚ùå Bad:   15/01/2026
‚ùå Bad:   01-15-2026
‚ùå Bad:   15-Jan-2026
‚ùå Bad:   Jan 15, 2026
```

**Why?**
- Sorts chronologically automatically
- Internationally recognized
- No ambiguity (MM/DD vs DD/MM)
- System-friendly

### Date + Time Format

**When timestamp needed:**
```
Format: YYYY-MM-DD_HHMM
Example: 2026-01-15_1430
```

### Date Ranges

**Format:** `YYYY-MM-DD_to_YYYY-MM-DD`
```
Example: 2026-01-01_to_2026-01-31
```

### Month Only

**Format:** `YYYY-MM`
```
Example: 2026-01
```

### Quarter Format

**Format:** `YYYY_QX`
```
Examples:
2026_Q1 (Jan-Mar)
2026_Q2 (Apr-Jun)
2026_Q3 (Jul-Sep)
2026_Q4 (Oct-Dec)
```

## üè† Property Naming

### Property ID Format

**Format:** `PROP-XXXX` (4 digits, zero-padded)

```
‚úÖ Good:
PROP-0001
PROP-0023
PROP-0456
PROP-1234

‚ùå Bad:
PROP-1
PROP01
property-001
P-0001
```

**Numbering System:**
- 0001-0999: Condos
- 1000-1999: Villas
- 2000-2999: Houses
- 3000-3999: Land
- 4000-4999: Commercial

### Property File Names

**Format:** `PROP-XXXX_Type_Size_Details.ext`

```
Examples:
PROP-0001_1BR_35SQM_Info.pdf
PROP-0001_1BR_35SQM_Floorplan.pdf
PROP-0025_Villa_250SQM_Contract.pdf
PROP-1234_Condo_Studio_28SQM.xlsx
```

### Property Photos

**Format:** `PROP-XXXX_Location_Number.jpg`

```
Examples:
PROP-0001_Bedroom_01.jpg
PROP-0001_Bedroom_02.jpg
PROP-0001_Kitchen_01.jpg
PROP-0001_Bathroom_01.jpg
PROP-0001_View_01.jpg
PROP-0001_Exterior_01.jpg
```

**Location Tags:**
```
Bedroom
Kitchen
Bathroom
Living_Room
Balcony
View
Exterior
Building
Pool
Gym
Common_Area
Parking
```

**Numbering:**
- Always 2 digits: 01, 02, 03...
- Start from 01 for each location
- Order by importance

### Property Folders

**Format:** `PROJECT_NAME_LOCATION`

```
‚úÖ Good:
THE_BASE_CENTRAL_PATTAYA
PARK_BEACH_JOMTIEN
ANANYA_RESIDENCE_NA_JOMTIEN

‚ùå Bad:
the base central

## üë• Lead Naming

### Lead ID Format

**Format:** `LEAD-YYYYMMDD-XXX` (3 digits, zero-padded)

```
‚úÖ Good:
LEAD-20260115-001
LEAD-20260115-002
LEAD-20260126-001

‚ùå Bad:
LEAD-001
LEAD-2026-01-15-1
L-20260115-001
```

**Components:**
- `LEAD-` prefix (required)
- `YYYYMMDD` date received (8 digits)
- `-` separator
- `XXX` sequence number (3 digits, starts fresh each day)

### Lead File Names

**Format:** `LEAD-YYYYMMDD-XXX_Name_Type.ext`

```
Examples:
LEAD-20260115-001_John_Smith_Profile.pdf
LEAD-20260115-001_John_Smith_Contract.pdf
LEAD-20260115-002_Anna_Johnson_Passport.jpg
LEAD-20260115-003_Somchai_Jaidee_Documents.zip
```

### Lead Notes Files

**Format:** `YYYYMMDD_Lead_Name_Notes.txt`

```
Examples:
2026-01-15_John_Smith_Meeting_Notes.txt
2026-01-16_Anna_Johnson_Call_Notes.txt
```

## üî¢ Version Control

### Version Numbering

**Format:** `_vX.Y` suffix

```
Examples:

```

**Version Rules:**
- `v1.0` ‚Üí Initial version
- `v1.1` ‚Üí Minor change (typo fix)
- `v2.0` ‚Üí Major change (content update)

### Date + Version

**For important documents:**

```

Contract_John_Smith_2026-01-16_v2.0.pdf
```

### Avoid Version Words

```
‚ùå Bad:

## üö´ What to Avoid

### Characters to NEVER Use

```
‚ùå Spaces (use _ instead)
‚ùå Special characters: / \ : * ? " < > |
‚ùå Dots (except for file extension)
‚ùå Brackets: [ ] ( )
‚ùå Ampersands: &
‚ùå Percent: %
‚ùå At: @
‚ùå Hash: #
```

### Bad Practices

```
‚ùå Vague names: "document.pdf", "file1.xlsx"
‚ùå Personal naming: "johns_property.pdf"
‚ùå Random naming: "image001.jpg", "scan0001.pdf"
‚ùå No dates: "report.pdf" (which month?)
‚ùå Inconsistent casing: "ProPeRtY_LiSt.xlsx"
‚ùå Long names: "This_is_a_very_long_filename_that_nobody_wants_to_type_ever.pdf"
```

### Length Limits

**Keep names under 50 characters:**

```
‚úÖ Good (35 chars):
2026-01-15_Meeting_John_Smith.pdf

‚ùå Too Long (78 chars):
2026-01-15_Meeting_Notes_With_John_Smith_Regarding_Property_Purchase_Discussion.pdf

‚úÖ Better:
2026-01-15_Meeting_John_Property_Discuss.pdf
```

## üîÑ Renaming Files

### How to Rename Safely

**DO NOT rename directly in Google Drive!**

Instead:
1. Download the file
2. Rename locally following convention
3. Upload with new name
4. Delete old file
5. Update any links/references

### Batch Renaming

**For multiple files:**
1. Make list of old ‚Üí new names
2. Share with team for review
3. Schedule rename (off-hours)
4. Rename all at once
5. Update documentation
